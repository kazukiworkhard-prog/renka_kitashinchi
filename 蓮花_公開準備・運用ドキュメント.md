## 5. WordPress 移行・設定について

現在のサイトは静的HTML（手書きコード）で制作されています。
クライアント自身がコンテンツを更新できるようにするため、WordPress への移行を推奨します。

### WordPress を選ぶ理由

| 項目 | 静的HTML（現在） | WordPress（移行後） |
|------|------------------|----------------------|
| コンテンツ更新 | 制作担当者のみ可能 | 店舗担当者が自分で更新可能 |
| メニュー変更 | HTMLを直接編集が必要 | 管理画面から操作可能 |
| お知らせ追加 | HTMLを直接編集が必要 | ブログ感覚で投稿可能 |
| 写真差し替え | ファイル差し替えが必要 | 管理画面からアップロード可能 |
| 費用 | サーバー代のみ | サーバー代 + WordPress（無料）|

### WordPress 移行の作業内容

**サーバー・環境構築**
- [ ] WordPress対応レンタルサーバーの契約（例：エックスサーバー、さくらインターネット）
- [ ] WordPress本体のインストール
- [ ] SSL証明書の設定（https化）
- [ ] ドメインとサーバーの紐付け設定

**テーマ・デザインの移行**
- [ ] 現在のHTMLデザインをWordPressテーマとして実装
- [ ] CSS・JavaScript・フォントの移行
- [ ] トップ動画・ギャラリー・各セクションの組み込み

**プラグインの設定**
- [ ] SEO対策：Yoast SEO または All in One SEO（メタ情報・構造化データの管理）
- [ ] セキュリティ：Wordfence（不正アクセス防止）
- [ ] バックアップ：UpdraftPlus（自動バックアップ）
- [ ] キャッシュ：WP Super Cache（表示速度の最適化）
- [ ] お問い合わせ：Contact Form 7（将来的にフォームを追加する場合）

**管理画面のカスタム設定**
- [ ] メニュー更新用のカスタムフィールド設定
- [ ] ギャラリー・お知らせのカテゴリ設定
- [ ] 管理画面を店舗担当者が使いやすいよう簡略化

**動作確認・納品**
- [ ] 全ページの表示・リンク確認
- [ ] スマートフォン・タブレット・PCでの表示確認
- [ ] 管理画面の操作説明（店舗担当者向けマニュアル作成）

## 6. スケジュール見積もり

### フェーズ1：静的HTMLサイトの公開（現在の状態）

| 作業 | 期間 |
|------|------|
| 残りコンテンツ（写真・テキスト）の準備 | 1〜2週間 |
| インフラ設定（ドメイン・サーバー・SSL） | 3〜5日 |
| 最終テスト・修正 | 3〜5日 |
| **公開まで合計** | **約3〜4週間** |

> コンテンツ（料理写真・テキスト）の準備が整い次第、インフラ設定と並行して進めることで短縮可能です。

### フェーズ2：WordPress 移行

| 作業 | 期間 |
|------|------|
| サーバー・WordPress環境構築 | 3〜5日 |
| テーマ（デザイン）の移行・実装 | 2〜3週間 |
| プラグイン設定・動作確認 | 3〜5日 |
| 操作マニュアル作成・納品 | 3〜5日 |
| **移行作業合計** | **約4〜5週間** |

### 全体スケジュール（目安）

```
3月下旬  写真・テキストの最終確認・準備
4月上旬  インフラ設定・最終テスト
4月中旬  静的HTMLサイト 公開 ← ここで一度公開
4月下旬  WordPress移行作業開始
5月下旬  WordPress版への切り替え・マニュアル納品
```

> WordPressへの移行は公開後でも行えます。まず静的サイトで公開し、並行して移行作業を進める方法が最もリスクが低くおすすめです。

## 8. 更新作業の依頼方法

サイトの更新が必要な際は、以下の情報を制作担当者にお伝えください。

**メニュー更新の場合**
- 料理名・説明文・価格
- 対象のコース名

**お知らせ更新の場合**
- 掲載日
- カテゴリ（Holiday / Menu / News など）
- 本文テキスト

**写真差し替えの場合**
- 差し替え箇所（ギャラリー何枚目、など）
- 画像ファイル（推奨：横1200px以上、JPG形式）
